# Mission Control Protocol Specification

Protocol name: **MCPS/1.0**

Transport: TCP

Encoding: UTF-8 compatible ASCII text lines

Line delimiter: `\n`; `\r\n` is accepted.

## Session lifecycle

A typical session is:

```text
CONNECT
AUTH mission-secret
GET_STATUS
SET_MODE ACTIVE
START_STREAM
STOP_STREAM
QUIT
```

`PING` and `HELP` are allowed without authentication. Operational commands require `CONNECT` and `AUTH` first.

## Command reference

### CONNECT

Initializes the protocol session.

Response:

```text
OK CONNECTED protocol=MCPS/1.0
```

### AUTH `<token>`

Authenticates the client.

Success:

```text
OK AUTHENTICATED role=operator
```

Failure:

```text
ERR AUTH_FAILED invalid token
```

### GET_STATUS

Returns the current mission/server status.

Example:

```text
OK STATUS mode=SAFE any_stream_active=false connected_clients=1 authenticated_clients=1 active_streams=0 commands=3 protocol_errors=0 auth_failures=0 telemetry_events=0 uptime_ms=420
```

### SET_MODE `SAFE|ACTIVE|MAINTENANCE`

Changes the global mission mode.

Example:

```text
OK MODE_SET mode=ACTIVE
```

### START_STREAM

Starts telemetry events for the current client session.

Example:

```text
OK STREAM_STARTED period_ms=500
EVENT TELEMETRY peer=127.0.0.1:54321 mode=ACTIVE seq=1 temperature_c=23.4 battery_pct=92.1 stream_active=true
```

### STOP_STREAM

Stops telemetry events for the current client session.

### PING

Liveness check.

```text
OK PONG uptime_ms=100
```

### HELP

Lists supported commands.

### QUIT

Requests clean session shutdown.

## Response classes

Responses always begin with one of:

- `OK`: command accepted and executed.
- `ERR`: command rejected or protocol error.
- `EVENT`: asynchronous event generated by the server.

## Error codes

Common errors:

- `EMPTY_COMMAND`
- `UNKNOWN_COMMAND`
- `BAD_ARGUMENT_COUNT`
- `UNEXPECTED_ARGUMENT`
- `INVALID_MODE`
- `NOT_CONNECTED`
- `NOT_AUTHENTICATED`
- `AUTH_FAILED`
